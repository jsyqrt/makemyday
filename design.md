这是一个日程规划的AI Web App。纯前端应用。静态js文件，没有后端服务器。

典型的使用场景是这样的。

用户打开这个Web App，按住一个按钮，开始说话。或者在一个居中的文本框中输入文本。
AI会把用户说的事情，分成具体可操作的事件，并对每个事件给出优先级和行动建议。
用户可以对生成的事件进行编辑，删除，拖拽排序，并最终生成一个日程规划。
优先级按照“紧急重要”四个象限来划分。
如果要增加事件，可以点+按钮，就会弹出文本输入框或者按住说话按钮。
该应用是一个开源应用，使用的ai需要用户自行配置llm api key和base url。另外，也需要配置语言识别的api。如果没有配置，则不支持语音识别。
存储直接放在浏览器中，如果用户要强制刷新，提醒可能会清掉数据。尽量不要清理缓存。

初次打开的时候，提醒配置llm api key 和 base url。以及语音识别的api代码。并提供测试功能。
缺省使用siliconflow的api，用户只需配置api key。
